# 多语言古籍专题页池及叙事单元提取系统使用教学文档

本文档面向项目交付后的实际使用人员，说明如何启动系统、配置模型、导入资料、发现与确认专题、生成页面池与叙事单元，并下载结果文件。

## 1. 系统用途

本系统用于对地方志、古籍影印 PDF 或已经整理好的 Excel 页面底表进行智能处理，形成可研究专题、专题页面池、叙事单元表和导出文件。

系统主流程为：

```text
上传 PDF / Excel
→ OCR 识别或页面导入
→ 发现与确认专题
→ 页面池评分
→ 叙事单元提取
→ 导出结果
```

系统目前默认跳过“页级分类”步骤，直接基于 OCR/Excel 底表内容进行多专题联合页面池评分，以减少重复调用大模型和降低成本。

## 2. 启动系统

### 2.1 Docker 全栈启动

在项目根目录双击或运行：

```bat
run.bat
```

系统会通过 Docker Compose 启动后端和相关服务。启动完成后，在浏览器访问：

```text
http://127.0.0.1:8000
```

### 2.2 本地开发模式启动

如果只启动数据库和本地后端，可按以下顺序：

1. 双击或运行 `start-db.bat`
2. 双击或运行 `run-local-backend.bat`
3. 打开浏览器访问 `http://127.0.0.1:8000`

### 2.3 启动后检查

页面顶部显示“本地 Docker 运行”时，说明前端界面已打开。

如果页面无法打开，可检查：

- Docker Desktop 是否已经启动
- 端口 `8000` 是否被占用
- 终端窗口是否出现后端报错
- 浏览器是否访问了正确地址

## 3. 界面区域说明

系统界面分为三栏：

### 3.1 左侧：资料输入

用于上传 PDF 或 Excel、配置模型、查看文档队列和处理进度。

主要功能：

- 选择上传类型：`PDF 古籍` 或 `Excel 页面集`
- 上传文件
- 设置 OCR 批次、处理并发、专题批次
- 配置 OCR 模型和文本处理模型
- 查看文档状态
- 查看处理流程进度

### 3.2 中间：智能体对话与专题

用于与 AI 讨论专题、批量提取专题、手动添加专题、勾选专题并启动后续处理。

主要功能：

- `发送给 AI`：与模型对话
- `AI 批量提取专题`：让系统从底表中自动发现专题
- `添加专题`：手动录入专题配置
- 专题列表：查看、勾选、删除专题
- `处理选中专题`：生成页面池、叙事单元和 ZIP 文件

### 3.3 右侧：文件输出

用于选择输出文件夹和下载结果。

主要功能：

- 选择保存文件夹
- 查看底表和专题结果 ZIP
- 下载或自动保存输出文件

## 4. 模型配置

系统分为两类模型：

### 4.1 OCR 模型

OCR 模型用于识别 PDF 图像页面中的文字。

点击左侧 `OCR 模型`，填写：

- 模型厂商
- API Key
- 模型名称
- 页面文字提示，选填
- OCR 提示词，通常保持系统默认

页面文字提示用于提醒 OCR 模型注意文字体系或语种，例如“繁体”“满汉合璧”“藏文夹注”等。该提示只作为识别线索，系统仍要求模型以页面图像实际内容为准，不翻译、不改写、不现代化。

### 4.2 文本处理模型

文本处理模型用于：

- AI 批量提取专题
- 智能对话
- 页面池评分
- 叙事单元提取

点击左侧 `文本处理模型`，填写：

- 模型厂商
- API Key
- 模型名称
- 文本处理提示词，通常保持系统默认

### 4.3 查询模型

在模型配置弹窗中，可点击 `查询模型` 获取当前厂商 API Key 可用的模型列表。如果查询失败，请检查：

- API Key 是否正确
- 厂商接口是否可访问
- 账号是否有对应模型权限

### 4.4 自定义提示词

模型配置弹窗中会显示当前提示词。默认情况下使用系统内置提示词。

如需自定义：

1. 点击提示词区域右上角 `+`
2. 编辑提示词
3. 点击确认保存

已保存的提示词会进入历史列表。后续可从列表中选择，也可以删除不用的历史提示词。

建议只有在确有明确任务需求时才修改提示词。随意改动提示词可能影响 OCR、专题发现或结构化提取质量。

## 5. 参数设置建议

### 5.1 OCR 批次

含义：一次提交给 OCR 模型处理的页数。

建议：

- 普通测试：`1`
- 模型支持多图输入且稳定时：可适当提高
- 如果 OCR 失败或超时：降低批次

### 5.2 处理并发

含义：页面池评分、叙事单元提取等文本处理阶段同时运行的模型请求数量。

建议：

- 小文件测试：`3-5`
- 中等规模：`5-20`
- 大规模生产：根据模型服务并发额度设置

注意：并发越高，速度可能越快，但也更容易触发模型服务限流、超时或费用快速增加。

### 5.3 专题批次

含义：AI 批量提取专题时，每批读取多少页底表内容。

建议：

- 小文件：`50-100`
- 大文件：`100-300`
- 如果专题过碎：可适当增加批次，并依赖最终汇总合并
- 如果模型上下文不足或报错：降低批次

系统会在最后汇总各批次候选专题，并要求模型合并相近专题、删除微专题。

## 6. 上传与 OCR / 页面导入

### 6.1 上传 PDF

1. 左侧选择 `PDF 古籍`
2. 点击上传区域选择 PDF
3. 在文档队列中点击 `执行 OCR`
4. 等待 OCR 完成

OCR 完成后，系统会生成底表：

```text
文件名_底表_每页OCR内容.xlsx
```

底表每行通常对应一页，包含页码和页面文字内容。

### 6.2 上传 Excel

1. 左侧选择 `Excel 页面集`
2. 上传 `.xlsx` 文件
3. 在文档队列中点击 `导入页面`
4. 等待导入完成

Excel 导入完成后，后续流程与 PDF OCR 生成的底表一致，都可以继续进行专题分析、页面池评分和叙事单元提取。

### 6.3 取消 OCR / 页面导入

文档处理中会显示 `取消` 按钮。

如果误触发 OCR 或导入，可点击取消。取消后可重新执行。

注意：已经提交给模型服务且正在响应中的单个请求可能不会瞬间中断，但系统会尽量停止后续任务继续执行。

## 7. 选择输出文件夹

在右侧 `文件输出` 区域点击 `选择文件夹`。

选择后，系统会把底表和后续专题 ZIP 自动保存到该文件夹。

建议在正式处理前先选择输出目录，避免任务完成后找不到文件。

如果浏览器不支持选择文件夹，系统会退回普通下载方式。建议使用新版 Chrome 或 Microsoft Edge。

## 8. AI 批量提取专题

OCR 或 Excel 导入完成后，可点击中间区域的：

```text
AI 批量提取专题
```

系统会：

1. 按专题批次读取底表页面
2. 每批提取候选专题
3. 汇总所有批次候选
4. 合并同义、近义、上下位或研究问题相近的专题
5. 生成最终专题列表

AI 批量提取出的专题包含：

- 专题名称
- 专题描述
- 核心词
- 扩展词
- 页面池对象
- 可抽取单元
- 可能回答的问题
- 证据页码
- 佐证摘录

这些字段会进入后续页面池评分和叙事单元提取。

### 8.1 停止专题提取

专题提取任务运行时，会出现 `停止专题提取` 按钮。

如果发现误触发、模型配置错误或费用风险较高，可点击停止。

### 8.2 查看专题证据

AI 批量提取完成后，聊天区会显示当前专题列表、证据页码和部分佐证摘录。

如果需要追问，可在对话框询问：

```text
这些专题分别有哪些页码和原文证据？
```

注意：AI 对话受上下文长度限制，不能保证一次看到全部底表内容。正式核验仍建议查看导出的底表和页面池文件。

## 9. 手动添加专题

点击中间区域 `添加专题`，系统会弹出手动添加专题表单。

手动添加不调用大模型，适合用户已经明确知道要研究什么专题的情况。

表单包括四项：

### 9.1 专题名称

必填。填写要研究的专题名称，建议简短明确。

### 9.2 专属字段

选填但推荐填写。

该项用于后续筛选页面池，可以填写：

- 关键词
- 栏目名
- 页面类型
- 对象名称
- 证据类型

可用顿号、逗号、分号、空格或换行分隔。

系统会把这些内容写入：

- 页面池对象
- 核心词

### 9.3 可抽取单元

选填。

该项用于决定后续叙事单元表中希望抽取哪些字段。可填写多个短字段名。

系统会把这些内容作为叙事单元表字段的一部分。

### 9.4 可能回答的问题

选填。

该项用于说明该专题希望支持回答的研究问题。可写一条或多条。

系统会将这些问题写入专题配置，用于辅助页面池评分和后续抽取。

## 10. 专题列表管理

专题列表显示在中间区域。

可执行操作：

- 勾选专题：决定后续是否处理
- 全选：勾选全部专题
- 全不选：取消全部专题
- 收起/展开：折叠或展开专题列表
- 删除：删除不需要的专题

删除专题是真删除。删除后该专题不会参与后续处理；如果该专题已经生成过配置、页面池或叙事结果，系统会同步删除相关后端专题配置。

正在处理选中专题时，不建议删除专题。系统会提示先停止处理再删除。

## 11. 处理选中专题

勾选要处理的专题后，点击：

```text
处理选中专题
```

系统会自动执行：

1. 创建专题配置
2. 多专题联合页面池评分
3. 按专题提取叙事单元
4. 生成专题结果 ZIP
5. 自动保存到输出文件夹

### 11.1 多专题联合页面池评分

如果一次勾选多个专题，系统会将多个专题一起放入同一轮页面池评分，让模型判断每页是否属于某个专题。

特点：

- 一页可以属于多个专题
- 不要求先做页级分类
- 分数低于阈值的专题不写入页面池
- 如果某专题本轮没有命中页面，不会覆盖旧页面池

如果一次选择的专题很多，系统会按每组最多 8 个专题拆分成多个联合评分任务。

### 11.2 叙事单元提取

页面池生成后，系统会针对每个专题的页面池提取叙事单元。

叙事单元表字段来源：

- 系统默认字段
- 手动专题中填写的“可抽取单元”
- AI 批量提取专题时生成的“可抽取单元”

### 11.3 停止处理

处理选中专题时，会出现：

```text
停止处理
```

可用于误触发、模型配置错误或费用风险较高时中止。

说明：

- 已经完成并保存的结果会保留
- 未开始的专题不会继续处理
- 已提交给模型服务的当前请求可能不会瞬间中断，但系统会停止后续批次和后续专题

## 12. 输出文件说明

### 12.1 底表

OCR 或 Excel 导入完成后生成。

常见文件名：

```text
原文件名_底表_每页OCR内容.xlsx
```

主要包含：

- 页码
- 页面内容

### 12.2 专题 ZIP

每个专题生成一个 ZIP。

ZIP 内通常包括：

- 专题页面池表
- 叙事单元表
- 专题配置文件
- 相关导出结果

### 12.3 页面池表

页面池表用于查看哪些页面被判定为与专题相关。

常见字段：

- 页码
- 总分
- 置信度
- 等级
- 命中关键词
- 文本内容
- 页面摘要

### 12.4 叙事单元表

叙事单元表用于进一步研究分析。

常见字段：

- 专题名称
- 叙事单元标题
- 单元类型
- 涉及对象
- 时间线索
- 地点线索
- 事件或行为
- 关键词命中
- 原文证据
- 来源页码

如果专题配置中提供了“可抽取单元”，表格字段会优先结合这些自定义字段。

## 13. 智能体对话使用建议

对话框适合用于：

- 请 AI 分析当前材料可能有哪些专题
- 询问某个专题是否有证据
- 追问专题之间的区别
- 讨论是否需要合并或删除专题

建议提问方式：

```text
请根据当前底表分析可研究专题，并说明每个专题的页码证据。
```

```text
这些专题中哪些比较接近，可以合并？
```

```text
请说明“水利灾荒”这个专题有哪些佐证页码和原文摘录。
```

注意：

- 对话上下文可能无法一次包含全部大文件底表
- AI 回答只能作为辅助判断
- 重要结论应结合底表、页面池和原文核验

## 14. 成本与效率建议

大模型调用费用通常由输入 tokens、输出 tokens、并发和任务规模决定。

较耗费 tokens 的阶段：

- OCR，尤其是 PDF 图像页
- AI 批量提取专题
- 页面池评分
- 叙事单元提取

节省成本建议：

- 先用小文件或少量页测试模型配置
- 确认 OCR 效果后再处理大文件
- 大文件先使用 AI 批量提取专题，再删除不需要的专题
- 只勾选确实要处理的专题
- 并发不要超过模型服务实际承受能力
- 发现模型配置错误时及时点击停止按钮

## 15. 常见问题

### 15.1 提示“请先配置模型”

说明 OCR 模型或文本处理模型尚未配置。

处理方式：

1. 点击左侧 `OCR 模型` 或 `文本处理模型`
2. 填写厂商、API Key、模型名
3. 点击确认

### 15.2 模型返回 401

通常表示 API Key 错误、过期或无权限。

处理方式：

- 检查 API Key 是否复制完整
- 检查模型厂商是否选错
- 检查账号是否开通对应模型
- 检查服务商余额或权限

### 15.3 OCR 很慢

可能原因：

- PDF 页数较多
- OCR 模型响应慢
- OCR 批次设置过大
- 网络或服务商限流

处理方式：

- 降低 OCR 批次
- 换用更快的 OCR 模型
- 分文件处理

### 15.4 专题提取太多

系统已在最终汇总阶段要求模型合并相近专题，但大文件仍可能产生较多专题。

处理方式：

- 删除不需要的专题
- 只勾选重点专题处理
- 在对话中要求 AI 判断哪些专题可合并

### 15.5 页面池没有命中页面

可能原因：

- 专题关键词或页面池对象过窄
- OCR 文本质量不佳
- 当前材料确实缺少相关内容

处理方式：

- 检查底表 OCR 内容
- 调整专题的专属字段或关键词
- 手动添加更贴近原文的页面池对象

### 15.6 叙事单元为空

可能原因：

- 页面池页面相关性不够
- 叙事字段设置过细
- 模型没有抽取到明确单元

处理方式：

- 查看页面池表，确认页面是否真的相关
- 减少或调整可抽取单元
- 重新处理该专题

### 15.7 文件没有自动保存

可能原因：

- 未选择输出文件夹
- 浏览器不支持文件夹写入
- 浏览器权限被拒绝

处理方式：

- 重新点击 `选择文件夹`
- 使用新版 Chrome 或 Edge
- 使用下载按钮手动保存

## 16. 使用注意事项

1. 大模型结果不是人工校勘结果，正式研究前需要核验原文。
2. OCR 可能出现错字、漏字、断行等问题，底表质量会影响后续所有结果。
3. 手动专题越明确，页面池评分越稳定。
4. 专属字段建议贴近原文词汇，不宜过于抽象。
5. 可抽取单元建议使用短字段名，不要写成长句。
6. 大文件处理前应先做小规模测试。
7. 模型 API Key 属于敏感信息，请妥善保管。
8. 若发现任务误触发，应及时点击停止按钮。

## 17. 推荐操作流程

正式处理一个新项目时，建议按以下顺序：

1. 启动系统并打开网页
2. 配置 OCR 模型和文本处理模型
3. 选择输出文件夹
4. 上传 PDF 或 Excel
5. 执行 OCR 或页面导入
6. 下载并抽查底表
7. 点击 AI 批量提取专题
8. 删除明显不需要的专题
9. 必要时手动添加重点专题
10. 勾选要处理的专题
11. 点击处理选中专题
12. 等待 ZIP 自动保存
13. 检查页面池表和叙事单元表
14. 如有需要，调整专题字段后重新处理

## 18. 交付验收建议

验收时可准备一个小型测试文件，按以下项目检查：

- PDF 上传是否正常
- OCR 是否能完成并生成底表
- Excel 底表是否能导入
- 模型配置是否可保存
- AI 批量提取专题是否能生成专题列表
- 手动添加专题是否能添加四项配置
- 专题删除是否生效
- 停止按钮是否可用
- 处理选中专题是否生成页面池
- 叙事单元表是否生成
- ZIP 是否能保存到指定文件夹

完成以上检查后，即可进入正式资料处理。
